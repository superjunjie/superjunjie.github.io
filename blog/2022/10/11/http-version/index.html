HTTP历史版本

Created2022-10-11|Updated2022-12-09|HTTP
|Post View:

HTTP/0.9

  1. 仅支持GET请求HTML文本

HTTP/1.0

  1. 支持多种类型文件的下载
  2. 引入了请求头和响应头，并key-value的形式保存信息
  3. 引入状态码
  4. 提供Cache机制

HTTP/1.1

  1. 增加持久连接。在一个TCP的基础上可以进行多次HTTP请求
  2. 管线化。为了解决头部阻塞问题，将多个HTTP请求合并提交给服务器
  3. 提供虚拟主机支持。请求头中加入Host字段，用来表示当前域名地址
  4. 引入 Chunk transfer 机制。对动态生成的内容提供了完美支持
  5. 提供cookie机制
  6. 可以给每个域名提供最多六个HTTP连接
  7. 支持CDN域名分片机制

主要问题

对带宽利用不理想。

  • TCP慢启动。TCP窗口拥塞控制策略导致TCP窗口总是从一个很小的数量开始再慢慢扩大
  • 开启多条TCP会竞争带宽，且无法设置请求优先级。导致应该被优先处理的请求得不到响应
  • 队头阻塞问题。一条TCP管道一次只能处理一个请求，在该请求响应之前，其他请求都会被阻塞

HTTP/2.0

多路复用机制
HTTP/2 的解决方案可以总结为：
一个域名只使用一个 TCP 长连接和消除队头阻塞问

HTTP/2 使用了多路复用技术，可以将请求分成一帧一帧的数据去传输，这样带来了一个额外的好处，就是当收到一个优先级高的请求时，比如接收到 JavaScript 或者 CSS 关键资源的请求，服务器可以暂停之前的请求来优先处理关键资源的请求
二进制分帧
通过引入二进制分帧层，就实现了 HTTP 的多路复用技术
在HTTP中加入二进制分帧层，二进制分帧可以将一个请求分为多个带有相同ID的请求帧，在服务器收到所以的请求帧后，会将这个带有相同ID是请求帧合并为一个完整的请求，会处理这个请求，并返回带有这个请求ID的响应内容。
设置请求的优先级
在HTTP/2在处理请求的时候，可以设置这个请求的优先级，当服务器收到很多个请求时，可根据优先级顺序处理这些请求
服务器推送
当用户请求一个HTML时，服务器知道该 HTML 页面会引用几个重要的 JavaScript 文件和 CSS 文件，那么在接收到 HTML 请求之后，附带将要使用的 CSS 文件和 JavaScript 文件一并发送给浏览器，这样当浏览器解析完 HTML 文件之后，就能直接拿到需要的 CSS 文件和 JavaScript 文件，这对首次打开页面的速度起到了至关重要的作用
头部压缩
HTTP/2对请求头和响应头进行压缩，提高传输效率

主要问题

  • TCP 建立连接的延时。一个TCP连接需要1.5个RTT
  • TCP协议僵化。主要是中间设备僵化，无法通过升级TCP协议解决
  • TCP队头拥塞。在将TCP分包传输后，如果前面的某个数据包没有收到或者丢失，那么整个TCP就只能暂停，等待丢失包重传

HTTP/3

基于UDP和QUIC协议
QUIC协议

  1. 实现了类似 TCP 的流量控制、传输可靠性的功能
  2. 集成TSL功能
  3. 实现多路复用功能
  4. 快速握手
