响应式对比

Vue2响应式原理

Vue2通过使用Object.defineProperty提供的get和set方法实现了对数据访问的监听以及对数据修改的劫持。由于Object.defineProperty的设计初衷不是为实现响应式设计的，所有使用这种方式在监听数据变化会存在下面这些问题或能力不足：

  1. 无法监听数组的变化，不支持使用下标直接修改数组内容，改变数组length属性无法被监听到。vue通过提供$set$del方法弥补了功能上的缺陷
  2. 无法监听新增属性以及已有属性删除
  3. 无法监听异质对象的变化，如Map，Set，WeakMap，WeakSet

Vue3响应式原理

通过使用Proxy+Reflect实现对数据的访问监听和修改劫持，相较于Vue2中的Object.defineProperty，Vue3中的响应式实现有下面这些优点

  1. 可以监听数组的变化
  2. 可以监听新增属性和删除属性
  3. 可以监听异质对象
  4. 支持包括get和set在内的13种数据方法访问监听

composition Vs options

options api

在options api中，业务逻辑被分散在各个模块，随着业务的复杂度越来越高，代码量会不断加大；由于相关业务的代码需要遵守option配置写到特定的区域，导致后续的维护变得非常复杂，代码的可复用性也不高。

composition api

使用composition api后，可以将相同业务逻辑放到一个函数中，业务代码之间的耦合度更高，更方便后续业务的扩展以及代码维护

Vue3相较于Vue2的优点

支持多根节点

相较于Vue2的每个template下面只能有一个根节点，vue3支持多根节点

支持异步组件（Supense）

支持Teleport

使用vue3天提供的telport内置组件，可将组件渲染到指定DOM节点下面

静态节点hoistStatic提升

在Vue2中，无论元素是否为静态节点，每次创建的时候都会被渲染。在vue3中对于是静态节点的元素，会进行变量提升，静态节点元素会被创建一次，在渲染的时候直接复用即可，对与下面要渲染的内容，vue3会以下处理

增加静态标记（pathFlags）

在创建vnode的时候，如果是动态节点，vue框架会识别出这个节点需要动态渲染内容的类型并标记patchflag。当视图更新时，只对动态节点部分进行diff运算，减少了资源的损耗

事件缓存cacheHandlers

在Vue2中，绑定事件每次触发都会生成全新的function去更新，cacheHandlers 是Vue3中提供的事件缓存对象，当 cacheHandlers 开启，会自动生成一个内联函数，同时生成一个静态节点。当事件再次触发时，只需从缓存中调用即可，无需再次更新。

Diff算法优化

vue2的diff算法是全量的对比，vue3中新增了静态标记patchFlags，在进行diff的时候，只对比带有pathflags的节点，还可以通过patchflags知道要对比的具体内容，从而减少了diff的次数，提升Diff算法的效率。

全面支持tree-shaking

在vue2中，全局的api都是暴漏在Vue实例上，即使没有使用，也无法通过tree-shaking进行消除。vue3对全局和内部的API进行重构时，增加了对tree-shaking的支持，如果在项目中未使用nexttick api，那么在打包的时候就会将未使用的api移除掉，最终获得最小打包体积。

全面支持TypeScript

Vue3使用TypeScript重写，相较于Vue2有更好的TypeScript支持

  • Vue2 Options API 中 option 是个简单对象，而 TypeScript 是一种类型系统，面向对象的语法，不是特别匹配
  • Vue2 需要vue-class-component强化vue原生组件，也需要vue-property-decorator增加更多结合Vue特性的装饰器，写法比较繁琐
